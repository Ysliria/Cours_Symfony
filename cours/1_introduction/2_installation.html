Installation

Prérequis

Symfony présente sur son site l'ensemble des prérequis à son utilisation :

  • PHP : dans une version supérieur ou égale à la 7.2.5.
  • Composer : le gestionnaire de dépendance de PHP.
  • Symfony CLI : la console de Symfony permet d'accélérer grandement la productivité ; elle va nous servir très rapidement dans le cours.
  • PHPStorm : ou un autre IDE avec lequel vous vous sentez à l'aise, car c'est avant tout VOTRE outil de travail ; il doit vous faciliter la vie !
  • GIT : un gestionnaire de version indispensable à TOUS les projets de développement.

Attention !

PHPStorm est payant. Vous bénéficiez d'un mois d'essai, voir plus en tant qu'étudiant. Néanmoins, passé ce délais il vous faudra vous acquittez du prix de la licence !

Mais l'essayer c'est l'adopter.

Contrôle des prérequis

Comme je l'ai noté plus haut, on va très vite utiliser la console Symfony. Aller dans votre terminal et saisissez : symfony check:requirements
Cette commande va vous permettre de controller les prérequis de Symfony. Voici un exemple de retour : 


    λ symfony check:requirements

    Symfony Requirements Checker
    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    > PHP is using the following php.ini file:
    E:\laragon\bin\php\php-7.4.1-Win32-vc15-x64\php.ini

    > Checking Symfony requirements:

    ..................WW.....WW..


    [OK]
    Your system is ready to run Symfony projects


    Optional recommendations to improve your setup
    ~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~~

    * a PHP accelerator should be installed
    > Install and/or enable a PHP accelerator (highly recommended).

    * realpath_cache_size should be at least 5M in php.ini
    > Setting "realpath_cache_size" to e.g. "5242880" or "5M" in
    > php.ini* may improve performance on Windows significantly in some
    > cases.

    * "memory_limit" should be greater than "post_max_size".
    > Set "memory_limit" to be greater than "post_max_size".

    * "post_max_size" should be greater than "upload_max_filesize".
    > Set "post_max_size" to be greater than "upload_max_filesize".

    Please check that both the console and the web server

Erreurs possibles et contrôles supplémentaires

Erreur à l'exécution de la commande Symfony
Essayez de l'executer avec les droits sudo.
Symfony signale une nouvelle version
Vous pouvez valider la demande de mise à jour de la console Symfony. Ensuite vous pouvez valider la version en saisissant symfony dans le terminal
Erreur sur les prérequis PHP
Vérifier la version de PHP executée avec la comande : php -v
Contrôler GIT
Pour contrôler la bonne installation de GIT exécutez la commande git --version
Contrôler Composer
Pour contrôler composer exécutez la commande composer -v
